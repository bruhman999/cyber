DOCKER & PODMAN — REFERENCE NOTES
==================================

WHAT IS DOCKER?
---------------
Docker is a containerization platform that packages applications and all
their dependencies into isolated containers. Containers are lightweight,
portable, and run the same way on any machine.

Think of it like:
  Virtual Machine  = full OS + your app (heavy, slow to start)
  Container        = just your app + its dependencies (lightweight, instant start)

DOCKER vs PODMAN
----------------
Docker    = industry standard, requires a background daemon (dockerd) running as root
Podman    = drop-in Docker replacement, daemonless, rootless by default
            Fedora ships Podman instead of Docker — preferred on Fedora

Commands are almost identical:
  docker run nginx    →    podman run nginx
  docker ps           →    podman ps
  docker build        →    podman build

WHAT IS DOCKER HUB?
-------------------
Docker Hub (hub.docker.com) is the official public registry for container images.
Think of it like GitHub but for containers.

  - Search for pre-built containers (nginx, postgres, pihole, etc.)
  - Pull images directly to your machine
  - Push your own custom images
  - Official images are verified and maintained by the software vendor
  - Community images are user-uploaded (check stars and pull count before using)

INSTALL DOCKER ON FEDORA
-------------------------
Fedora recommends Podman but if you want Docker:

# Remove old versions
sudo dnf remove docker docker-client docker-client-latest docker-common \
     docker-latest docker-latest-logrotate docker-logrotate docker-engine

# Add Docker repo
sudo dnf config-manager --add-repo https://download.docker.com/linux/fedora/docker-ce.repo

# Install Docker
sudo dnf install docker-ce docker-ce-cli containerd.io docker-compose-plugin -y

# Start and enable Docker
sudo systemctl start docker
sudo systemctl enable docker

# Run without sudo (add your user to docker group)
sudo usermod -aG docker neo
newgrp docker

INSTALL PODMAN ON FEDORA (EASIER — ALREADY INSTALLED)
------------------------------------------------------
# Podman is pre-installed on Fedora, verify:
podman --version

# If not installed:
sudo dnf install podman -y

# Podman Desktop (GUI) optional:
sudo dnf install podman-desktop -y

CORE DOCKER / PODMAN COMMANDS
------------------------------
# Pull an image from Docker Hub
docker pull nginx
podman pull nginx

# Run a container
docker run nginx
podman run nginx

# Run in background (detached)
docker run -d nginx
podman run -d nginx

# Run with port mapping (host:container)
docker run -d -p 8080:80 nginx
podman run -d -p 8080:80 nginx
# Now access at http://localhost:8080

# Run with a name
docker run -d --name mywebserver -p 8080:80 nginx
podman run -d --name mywebserver -p 8080:80 nginx

# List running containers
docker ps
podman ps

# List ALL containers including stopped
docker ps -a
podman ps -a

# Stop a container
docker stop mywebserver
podman stop mywebserver

# Remove a container
docker rm mywebserver
podman rm mywebserver

# List downloaded images
docker images
podman images

# Remove an image
docker rmi nginx
podman rmi nginx

# View container logs
docker logs mywebserver
podman logs mywebserver

# Enter a running container (like SSH into it)
docker exec -it mywebserver bash
podman exec -it mywebserver bash

# Remove all stopped containers
docker container prune
podman container prune

VOLUMES — PERSISTENT STORAGE
-----------------------------
By default containers lose all data when stopped. Use volumes to persist data.

# Mount a local folder into the container
docker run -d -p 8080:80 -v /home/neo/webfiles:/usr/share/nginx/html nginx
podman run -d -p 8080:80 -v /home/neo/webfiles:/usr/share/nginx/html nginx

# Format: -v /host/path:/container/path

ENVIRONMENT VARIABLES
---------------------
Pass config into containers without editing files inside them.

docker run -d \
  -e POSTGRES_USER=neo \
  -e POSTGRES_PASSWORD=mypassword \
  -e POSTGRES_DB=mydb \
  -p 5432:5432 \
  postgres

DOCKER COMPOSE
--------------
Compose lets you define and run multi-container setups in one file.
Instead of long docker run commands, write a docker-compose.yml

Example docker-compose.yml (nginx + postgres):

  version: "3"
  services:
    web:
      image: nginx
      ports:
        - "8080:80"
      volumes:
        - ./webfiles:/usr/share/nginx/html

    db:
      image: postgres
      environment:
        POSTGRES_USER: neo
        POSTGRES_PASSWORD: mypassword
        POSTGRES_DB: mydb
      volumes:
        - pgdata:/var/lib/postgresql/data

  volumes:
    pgdata:

# Start everything
docker compose up -d

# Stop everything
docker compose down

# Install compose on Fedora if needed
sudo dnf install docker-compose-plugin -y

PORTAINER — DOCKER GUI (RECOMMENDED FOR HOME LAB)
--------------------------------------------------
Portainer gives you a web UI to manage all your containers visually.

# Run Portainer
docker run -d \
  --name portainer \
  -p 9443:9443 \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v portainer_data:/data \
  portainer/portainer-ce

# Access at: https://localhost:9443

PODMAN-SPECIFIC TIPS FOR FEDORA
--------------------------------
# Podman doesn't need sudo for most operations
podman run -d -p 8080:80 nginx

# Enable podman socket (needed for some tools expecting Docker socket)
systemctl --user enable podman.socket
systemctl --user start podman.socket

# Auto-start containers on boot (Fedora/systemd)
podman generate systemd --name mycontainer --files --new
sudo mv container-mycontainer.service /etc/systemd/system/
sudo systemctl enable container-mycontainer.service

# Podman Desktop — GUI for Podman (like Docker Desktop)
sudo dnf install podman-desktop -y
