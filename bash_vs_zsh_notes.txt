================================================================================
                    SHELL REFERENCE NOTES — BASH, ZSH, KSH & MORE
================================================================================
OVERVIEW
--------
A shell is the program that reads your commands and runs them in the terminal.
Linux/WSL can have multiple shells installed. The most common are Bash, Zsh,
and Ksh. They look similar but are different programs with different syntax and
config files. Running the wrong shell for your config causes errors.

Your setup:
  Default shell (wrong one launching): Bash  — shown by: echo $SHELL -> /bin/bash
  Correct shell for your config:       Zsh   — launched manually with: zsh
  Kali Linux default shell:            Zsh
================================================================================
                              WHAT EACH SHELL IS
================================================================================
BASH (Bourne Again Shell)
  - The default shell on most Linux systems
  - Ships on virtually every Linux distro, always available
  - Reliable and minimal — no frills, no themes
  - Config file: ~/.bashrc
  - Does NOT understand Zsh-specific commands (setopt, bindkey, zstyle, etc.)

ZSH (Z Shell)
  - Built on top of ideas from Bash, Ksh, and Csh — combines the best of each
  - Supports plugins, custom themes, better tab-completion, prompt customization
  - Default shell on macOS (since 2019) and Kali Linux
  - Config file: ~/.zshrc
  - Powers the fancy prompt you see on Kali (Oh My Zsh theme)
  - Best choice for daily use on modern Linux desktops

KSH (Korn Shell)
  - Developed at AT&T Bell Labs in the 1980s by David Korn
  - Designed as a superset of the original Bourne shell (sh)
  - Very POSIX compliant, lightweight, and reliable
  - Common in enterprise environments, legacy Unix, AIX, and Solaris systems
  - Many older corporate shell scripts were written specifically for Ksh
  - Config file: ~/.kshrc
  - Two main versions: ksh88, ksh93 (ksh93 is current), and mksh (MirBSD Korn Shell)
  - Install on Kali/Debian:  sudo apt install ksh
  - Install on Fedora:       sudo dnf install ksh

FISH (Friendly Interactive Shell)
  - Modern shell focused on user-friendliness out of the box
  - Has syntax highlighting, autosuggestions, and tab-completion with zero config
  - Does NOT follow POSIX standards — fish scripts are NOT compatible with
    bash/sh scripts, which can cause problems in some environments
  - Config file: ~/.config/fish/config.fish
  - Good for beginners who want a nice experience without setup work
  - Install on Kali/Debian:  sudo apt install fish
  - Install on Fedora:       sudo dnf install fish

DASH (Debian Almquist Shell)
  - Extremely lightweight and fast
  - Used as /bin/sh on Debian and Ubuntu — it is what runs your system scripts
  - Strictly POSIX — no interactive features, no themes, no extras
  - Not meant for daily interactive use, but very important under the hood
  - If a script runs with #!/bin/sh it is likely running Dash on Debian systems

TCSH / CSH (TENEX C Shell / C Shell)
  - Syntax based on the C programming language instead of Bourne shell syntax
  - Was popular in the 1980s-90s, especially on BSD systems
  - Mostly considered outdated now — avoid writing new scripts in Csh/Tcsh
  - Still available and sometimes seen on older Unix systems
  - Config file: ~/.tcshrc or ~/.cshrc

SH (Bourne Shell)
  - The original Unix shell, ancestor of Bash, Ksh, and Dash
  - On modern Linux, /bin/sh is usually a symlink to Bash or Dash, not true sh
  - Scripts written for sh are the most portable across all Unix-like systems
  - Not used interactively — only relevant for writing portable scripts

================================================================================
                         SHELL COMPARISON AT A GLANCE
================================================================================

  Shell   | Interactive Use | Scripting  | POSIX  | Modern Features
  --------|-----------------|------------|--------|------------------
  Bash    | Good            | Excellent  | Yes    | Moderate
  Zsh     | Excellent       | Excellent  | Mostly | Best
  Ksh     | Moderate        | Excellent  | Yes    | Minimal
  Fish    | Excellent       | Limited*   | No     | Best
  Dash    | Poor            | Good       | Yes    | None
  Tcsh    | Moderate        | Poor       | No     | Minimal

  * Fish scripts use their own syntax, not compatible with bash/sh

================================================================================
                            WHY SOURCING .ZSHRC IN BASH BROKE
================================================================================
WHAT HAPPENED
  1. Your terminal was launching Bash even though your config (.zshrc) is
     written for Zsh
  2. Running: source ~/.zshrc inside Bash told Bash to read a Zsh config file
  3. Bash does not know Zsh-only commands like setopt, bindkey, zstyle, autoload
  4. Result: hundreds of errors — "command not found", "bad substitution", etc.
  5. Your prompt also appeared stuck/wrong because the theme couldn't load

THE FIX
  Simply running: zsh
  ...launched the correct shell, which knows how to read .zshrc properly.
  Everything loaded as expected and the prompt updated correctly.

================================================================================
                              MAKING ZSH PERMANENT
================================================================================
Right now you have to type 'zsh' manually every time you open a terminal.
To make Zsh your default shell permanently:

  chsh -s $(which zsh)

Then close and reopen your terminal. It will launch Zsh automatically from
that point on and you will never need to type 'zsh' again.

To verify your default shell was changed:
  echo $SHELL
  -> should now return /bin/zsh or /usr/bin/zsh

The same command works for any shell — just swap zsh for ksh, fish, etc:
  chsh -s $(which ksh)
  chsh -s $(which fish)

Make sure the shell is listed in /etc/shells first or chsh will reject it:
  cat /etc/shells
  echo $(which ksh) | sudo tee -a /etc/shells   <- adds it if missing

================================================================================
                              QUICK COMMAND REFERENCE
================================================================================
  echo $SHELL              check what shell is currently set as default
  echo $0                  check what shell is actually running right now
  zsh                      manually launch Zsh in current session
  bash                     manually launch Bash in current session
  ksh                      manually launch Ksh in current session
  fish                     manually launch Fish in current session
  chsh -s $(which zsh)     set Zsh as your permanent default shell
  chsh -s $(which ksh)     set Ksh as your permanent default shell
  chsh -s $(which fish)    set Fish as your permanent default shell
  source ~/.zshrc          reload Zsh config (run inside Zsh, not Bash)
  source ~/.bashrc         reload Bash config
  source ~/.kshrc          reload Ksh config
  cat ~/.zshrc             view your Zsh config file
  cat /etc/shells          list all valid shells installed on the system
  which zsh                show the full path to the Zsh binary
  which ksh                show the full path to the Ksh binary

--------------------------------------------------------------------------------
SHELLS — BASH VS ZSH (WHY $SHELL STILL SHOWS BASH)
--------------------------------------------------------------------------------

PROBLEM
--------

You run:
  zsh

But then:
  echo $SHELL

Output:
  /bin/bash

This makes it look like you are still using bash.

--------------------------------------------------------------------------------
WHY THIS HAPPENS
--------------------------------------------------------------------------------

  $SHELL shows your DEFAULT LOGIN SHELL,
  NOT the shell you are currently running.

So:
  - You can be inside zsh
  - But your default shell is still bash

--------------------------------------------------------------------------------
HOW TO CHECK YOUR CURRENT SHELL (IMPORTANT)
--------------------------------------------------------------------------------

  echo $0

Output:
  zsh        → You are in zsh
  bash       → You are in bash

--------------------------------------------------------------------------------
MAKE ZSH YOUR DEFAULT SHELL
--------------------------------------------------------------------------------

  chsh -s /bin/zsh

Then restart your terminal OR run:
  exit

--------------------------------------------------------------------------------
VERIFY IT WORKED
--------------------------------------------------------------------------------

  echo $SHELL

Expected:
  /bin/zsh

--------------------------------------------------------------------------------
WSL NOTE (IMPORTANT)
--------------------------------------------------------------------------------

In WSL, chsh may not always work properly.

If it does NOT stick, use this workaround:

1. Edit bash config:
   nano ~/.bashrc

2. Add this line at the bottom:
   exec zsh

3. Save and restart terminal

This forces zsh to launch automatically.

OPTIONAL: FORCE FOR A SPECIFIC USER
--------------------------------------------------------------------------------

  sudo chsh -s /bin/zsh neo
